# Проект "MAKE_BANK"

## Описание:
Проект "MAKE_BANK" представляет собой приложение для управления банком, которое позволяет пользователям выполнять различные операции, такие как открытие и закрытие счетов, пополнение баланса, перевод средств и просмотр истории транзакций. Основная цель приложения - предоставить удобный и безопасный интерфейс для работы с финансовыми операциями.

## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/iPavels/make_bank.git
```
2. Использование функций:
```
get_mask_card_number, get_mask_account, filter_by_state, sort_by_date, mask_account_card, get_date
```
## Использование:
```
1. get_mask_card_number - Функция принимает на вход номер карты и возвращает ее маску.
2. get_mask_account - Функция принимает на вход номер счета и возвращает его маску.
3. filter_by_state - Принимает список словарей и значение для ключа state.
4. sort_by_date - Принимает список словарей и еще один параметр и сортирует. 
5. mask_account_card - Функция принимает строку с типом и номером карты или счета, возвращает замаскированный номер.
6. get_date - Функция возвращает дату.
```

# Генераторы в проекте

filter_by_currency(transactions, currency_code)
Генератор, который фильтрует список транзакций и возвращает только те, где код валюты соответствует заданному.

transaction_descriptions(transactions)
Генератор, который поочередно возвращает описания транзакций из списка.

card_number_generator(start, stop)
Генератор, который создает номера банковских карт в формате XXXX XXXX XXXX XXXX в заданном диапазоне от start до stop.

## Декораторы

В проекте реализован модуль `decorators`, содержащий логирующий декоратор `log`.

### `@log(filename: Optional[str] = None)`

Декоратор `log` автоматически логирует выполнение обёрнутой функции:

- начало и завершение выполнения;
- имя функции и результат при успехе;
- имя функции, тип ошибки и аргументы при исключении.

**Аргумент:**
- `filename` — необязательный путь к файлу:
  - если указан, лог сохраняется в файл;
  - если не указан, лог выводится в консоль.

## Новая функциональность

- В проекте реализован поиск транзакций по описанию с использованием регулярных выражений (`process_bank_search`) и подсчёт количества операций по заданным категориям (`process_bank_operations`).

- Также добавлена функция `main`, которая связывает всю логику проекта: загрузку данных, фильтрацию по статусу, сортировку по дате, выборку только рублёвых операций и поиск по описанию.
